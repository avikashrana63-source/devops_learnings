What is Cron Job?
Cron job = scheduled task (automatic run at specific time)

Example:
Daily backup at 2 AM
Restart service every night
Run script every 5 minutes
___________________________________________________________________________________________

Why it is used?
Automation (no manual work)
Regular tasks (backup, cleanup)
Monitoring scripts
DevOps pipelines
___________________________________________________________________________________________

Crontab Command :

1.1 Open cron editor (crontab -e)
1.2 View cron jobs (crontab -l)
1.3 Delete cron jobs (crontab -r)
___________________________________________________________________________________________

Cron Job Format:
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of week (0-7) (Sunday=0)
│ │ │ └──── Month (1-12)
│ │ └────── Day of month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
Easy Memory Trick
Minute → Hour → Day → Month → Week
__________________________________________________________________________________________

Examples:

1.1 (* * * * * echo "Hello" >> file.txt) Run every minute
1.2 (0 2 * * * /home/user/backup.sh) Run every day at 2 AM
1.3 (*/5 * * * * script.sh) Run every 5 minutes
1.4 (0 0 * * 0 script.sh) Run every Sunday
1.5 (30 10 * * * script.sh)  Run at 10:30 AM
___________________________________________________________________________________________

Real DevOps Use Cases:
 1. Backup every day (0 1 * * * /home/user/backup.sh)
 2. Clean logs (0 0 * * * rm -rf /var/log/*.log)
 3. Restart service (0 3 * * * systemctl restart nginx)
 4. Health check script (*/2 * * * * /home/user/health.sh)
